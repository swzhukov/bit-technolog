## 1. ЦЕННОСТИ ПРОДУКТА (принципы, на которых строим)

| Ценность | Что значит | Как проявляется |
|---|---|---|
| **Скорость входа** | Технолог видит результат за минуты, а не за дни | Latency ≤ 30 сек, MVP за 4-5 нед |
| **Low-entry** | Порог входа — не миллионы, а десятки тысяч ₽ | 5-20К₽/мес за пользователя vs 1,5М+ у конкурентов |
| **Вертикаль** | Специализация на спецтехнике, не горизонтальный AI | Шаблоны, справочники, правила под КиВТ |
| **Прозрачность** | AI объясняет, почему, а не «вот результат» | «Обоснование» для руководства + цитаты в warnings + источник для каждой duration |
| **On-premise** | Данные клиента не уходят наружу | Docker Compose на сервере заказчика, без интернета |
| **Человек решает** | AI — помощник, не замена | Технолог утверждает ТК, AI только предлагает |
| **Данные → продукт** | Каждый фидбек технолога улучшает систему | RAG обновляется на правках |
| **Сначала работает, потом красиво** | Не идеально, но работает | MVP v0.1 без интеграции с 1С, потом v0.2 с JSON |

## 2. ЦЕЛИ (зачем делаем)

### 2.1. Стратегическая (12-18 мес)

**Стать стандартом Бита по ускорению техкарт в 1С:ERP для мелкосерийного машиностроения.**

Проверка: 3+ клиента Бита с мелкосерийкой используют в проде + очередь 2-3 на пилот.

### 2.2. Тактическая (6-9 мес)

**Запустить коммерческий проект по КиВТ у Техинкома и довести AI-помощника технолога до работающего MVP в проде.**

Проверка: технолог Техинкома регулярно использует, время 6-8ч → 1-2ч, есть 1-2 кейса «без MVP не справились бы».

### 2.3. Операционные

| Горизонт | Цель |
|---|---|
| **3 мес** | MVP v0.1 работает, 1-3 ТК сделаны через MVP |
| **6 мес** | 3-5 типов изделий Техинкома, время 6-8ч → 1-2ч, точность 70%+, 1 кейс |
| **12 мес** | 3+ пилота в других клиентах Бита, продукт окупает себя |

### 2.4. Анти-цели (чего НЕ делаем)

- ❌ Не становимся горизонтальным AI-ассистентом для всех заводов
- ❌ Не конкурируем с ИИ-Технологом лоб в лоб (1,1М vs наш low-entry)
- ❌ Не делаем CAM/PDM/PLM
- ❌ Не делаем учёт трудозатрат для ГОЗ
- ❌ Не заменяем технолога — помогаем ему

## 5. ЧТО НЕ ВХОДИТ В MVP (явно)

- ❌ Генеративное проектирование (CAD-уровень)
- ❌ ЧПУ-программирование (CAM)
- ❌ Полноценная PDM (workflow согласования с ЭЦП — только имитация)
- ❌ Учёт оснастки и инструмента
- ❌ Интеграция с бухгалтерией и складом
- ❌ Multi-tenant
- ❌ Загрузка рабочих центров (мёртвое дитя для мелкой серии)
- ❌ Углублённое нормирование до детали
- ❌ Учёт трудозатрат для ГОЗ
- ❌ Автоматический отчёт для руководства (только базовые метрики)
- ❌ Управление изменениями КД (только уведомление об изменении)
- ❌ Распознавание 3D-геометрии (только свойства из файла)
- ❌ Мобильное приложение (только web)
