## 1. НАЗНАЧЕНИЕ

**БИТ.Технолог** — AI-помощник технолога для ускорения создания техкарт в 1С:ERP.

**Пилотный клиент:** ООО «ПК Техинком-Центр» (пожарная спецтехника).

**MVP v0.1:** принимает .m3d/.cdw от конструкторов, генерирует черновик техкарты, технолог правит и записывает в 1С:ERP как ресурсную спецификацию.

**Принцип:** 1С:ERP — мастер НСИ. Наша система только читает, кеширует, обогащает AI-метаданными. Не дублирует мастер-данные.

## 4. АРХИТЕКТУРА (краткая ссылка)

**Полная:** `bit-technolog-architecture.md`.

**Основные компоненты:**
1. **Watcher** — следит за .m3d/.cdw, парсит, публикует в очередь
2. **1С Connector** — OData/HTTP клиент для НСИ, кеш, запись РС
3. **LLM-агент** — генерирует черновик ТК через Claude
4. **RAG Index** — ChromaDB для поиска аналогов
5. **Web UI** — FastAPI + HTMX для технолога
6. **PostgreSQL** — наша БД (только AI-метаданные)

**Поток данных:**
```
КОМПАС → Watcher → Queue → LLM-агент → PostgreSQL → Web UI
                              ↑
1С:ERP → 1С Connector (кеш) ─┘
                              ↓
                         (write RS) → 1С:ERP
```

## 9. РИСКИ И МИТИГАЦИЯ

| Риск | Вероятность | Митигация |
|---|---|---|
| Нет доступа к 1С OData | Средняя | Fallback: прямой SQL к PostgreSQL 1С через прослойку |
| КОМПАС API не работает на Linux | Высокая | Fallback: парсинг PDF/DXF (OCR) — как у ИИ-Технолога |
| Latency LLM > 30 сек | Низкая | Оптимизация промта, кеширование, fallback на меньшую модель |
| Точность LLM < 70% | Средняя | Больше Few-shot, итерация промта, RAG на большей базе |
| Нет .m3d/.cdw от Техинкома | Средняя | Использовать тестовые детали от конструкторов |
