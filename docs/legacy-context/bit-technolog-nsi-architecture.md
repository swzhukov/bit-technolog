# БИТ.Технолог — 1С:ERP как мастер-база НСИ

> **Дата:** 2026-07-16
> **Назначение:** описание архитектуры, где **1С:ERP — единственный источник правды по НСИ**. Наша система только читает + обогащает.
> **Контекст:** обновление к `bit-technolog-context-v6.md` и `bit-technolog-requirements.md`.

---

## 0. ГЛАВНЫЙ ПРИНЦИП

**Вся НСИ (нормативно-справочная информация) живёт в 1С:ERP. Наша система БИТ.Технолог — НЕ дублирует мастер-данные, а только:**
1. **Читает** (через OData / HTTP-сервис)
2. **Кеширует** (для скорости)
3. **Обогащает** AI-метаданными (но не перезаписывает мастер-данные)
4. **Записывает результат** (ТК обратно в 1С:ERP)

**Никаких собственных справочников оборудования, материалов, операций. Всё в 1С.**

- Источник данных (например, «Материал: Сталь 09Г2С (из 1С:ERP, версия кеша 16.07.2026 14:30)»)

- **«В 1С»** → не экспорт в Excel, а **прямая запись РС в 1С через коннектор**

## 6. АРХИТЕКТУРА СИНХРОНИЗАЦИИ

### 6.1. Стратегия синхронизации

**Pull-based по расписанию + push-based по событию.**

| Сущность | Стратегия | Частота |
|---|---|---|
| Справочник оборудования | Pull + push | Раз в час + по событию |
| Справочник материалов | Pull + push | Раз в сутки + по событию |
| Справочник номенклатуры | Pull + push | Раз в час + по событию |
| Справочник подразделений | Pull | Раз в сутки (редко меняется) |
| Правила технолога | Pull + push | По событию |
| РС (для аналогов) | Pull | Раз в сутки (полный) + инкрементально по событию |

### 6.2. Формат обмена

**Чтение:** OData (стандарт 1С) или HTTP-сервис с JSON-ответом.
**Запись:** HTTP-сервис с JSON-телом (POST) для создания РС.

Пример запроса на чтение номенклатуры:
```
GET /odata/standard.odata/Catalog_Номенклатура?$filter=...
Headers: Authorization: Basic <base64>
```

Пример записи РС:
```
POST /api/rs/create
Body: {
  "nom_ref": "uuid-номенклатуры",
  "operations": [...],
  "ai_metadata": {"confidence": 0.85, "source": "аналог КРН-001-АЦ8"}
}
```

### 6.3. Что попадает в кеш

**В кеш:** все справочники целиком (они небольшие, десятки/сотни записей).
**Вне кеша:** документы (РС, заказы) — читаются по запросу.

### 6.4. Что попадает в AI-метаданные (наша таблица)

```sql
CREATE TABLE ai_metadata (
  rs_id UUID,                    -- ID РС в 1С
  confidence_overall FLOAT,
  duration_source TEXT,           -- JSON: {"op_015": "ГОСТ", "op_020": "аналог"}
  warnings_count INT,
  questions_count INT,
  generated_at TIMESTAMP,
  human_edited BOOLEAN,
  human_edit_log JSON
);
```
