## ЧАСТЬ 1. ПРОДУКТ

### 1.1. Что это

**AI-помощник технолога для ускорения создания техкарт в 1С:ERP** для мелкосерийного машиностроения с индивидуальными заказами.

**НЕ:**
- ❌ AI-надстройка для КИВТ
- ❌ AI-ассистент внедрения 1С:ERP
- ✅ Помощник технолога

### 1.2. Пилотный клиент

**ООО «ПК Техинком-Центр»** (г. Тверь) — пожарная спецтехника (АЦ, ПСС, МТБ, УМП) на шасси КАМАЗ/ГАЗ/УРАЛ/МАЗ/ЗИЛ.

### 1.3. Целевой пользователь

**Главный технолог** (Баранов М.А.) и 2-3 технолога Техинкома.

### 1.4. MVP/Прототип — что показываем

**Прототип:** технолог заходит → видит список деталей → кликает → получает черновик ТК за 30 сек → может править → экспортирует в Excel/PDF.

**5-10 mock-деталей** в JSON (имитация того, что приходит от конструкторов из КОМПАС-3D).

## ЧАСТЬ 4. КОНКУРЕНТ — ИИ-Технолог

**Что умеет:** распознавание чертежа (PDF, **не 3D**), нормы по ГОСТ, маршрут, спецификация, ТК, себестоимость, аналоги, проверка оборудования.
**Цены:** ППО 180К, лицензия от 1,1М, внедрение 1,5-2М, железо 0,5-1М. Итого вход 1,3-2,5М+.
**Точность:** 88-95% после обучения.
**Архитектура:** оркестр моделей + роботы.

**Наши преимущества:**
- 3D (мы — КОМПАС)
- Low-entry (5-20К/мес vs 1,5М+)
- Вертикаль спецтехники
- Быстрый старт (1-2 нед vs 3-4 мес)
- Стоимость MVP в 100-1000 раз дешевле

- Префикс **«ты — лучший в мире [роль]»** для промтов
- «Покритикуй», «задавай вопросы»
- Спорить и давать прямой взгляд
- **Фактология > гипотезы**
- **Без hedging**
- Структурированный вывод (таблицы, карточки)

### Чего НЕ делать
- ❌ Уходить в сторону
- ❌ Делать широкие обзоры без запроса
- ❌ «Отличный вопрос», «спасибо за уточнение»
- ❌ Длинные вступления
- ❌ Додумывать имена без grep'а
- ❌ Распыляться на уточнения, не влияющие на продукт
- ❌ Врать про усилия

### Формат
- Файлы в `/workspace/`
- Артефакты через `<deliver-assets>`
- По делу, без воды

## ЧАСТЬ 12. КАК ЗАГРУЗИТЬ В MiniMax Code

### Вариант 1. Web-интерфейс
1. Открыть https://agent.minimax.io/
2. Создать аккаунт (бесплатный тариф)
3. Создать проект
4. Загрузить файлы:
   - `bit-technolog-prototype-context.md` (этот файл)
   - `bit-technolog-prototype-tasks.md`
   - `bit-technolog-prototype-data.json`
   - `bit-technolog-prototype-prompt.txt`
5. Написать задачу: **«Реализуй прототип БИТ.Технолог по этому контексту»**

### Вариант 2. Desktop app
1. Скачать MiniMax Code Desktop (https://agent.minimax.io/download)
2. Установить (macOS или Windows)
3. Открыть проект
4. Загрузить файлы + задачу

### Вариант 3. API (для автоматизации)
- Anthropic-compatible endpoint: `https://api.minimax.io/anthropic`
- OpenAI-compatible endpoint: `https://api.minimax.io/v1`
- API Key: https://platform.minimax.io → API Keys
- Модели: MiniMax-M3 (1M context, рекомендуется), M2.7, M2.5

### Команда для MiniMax Code
```
Создай прототип веб-приложения БИТ.Технолог по контексту в файле
bit-technolog-prototype-context.md. Это FastAPI-приложение с HTMX-фронтендом,
которое демонстрирует AI-помощника технолога. Используй стек: Python 3.11+,
FastAPI, Jinja2, HTMX, SQLite, anthropic SDK. Mock-данные в файле
bit-technolog-prototype-data.json. Промт в файле bit-technolog-prototype-prompt.txt.
Задачи — в bit-technolog-prototype-tasks.md.
```
