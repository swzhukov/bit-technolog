# Руководство пользователя по 4 ролям

> **Версия:** v0.4.18 (2026-07-20) — M22: полный редизайн UI
> **Для кого:** все пользователи БИТ.Технолог
> **Главный принцип:** система **визуально подсказывает**, что вы можете делать в текущей
> роли. Если кнопки нет — значит, для вашей роли это действие недоступно.

---

## Как войти в систему

1. Откройте в браузере `http://ваш-сервер:8081/` (например, `http://localhost:8081/` для локальной разработки)
2. Если у вас есть логин/пароль — введите их (на пилоте не используется, см. ниже)
3. Если вы на демо или пилоте — **выберите роль** в правом верхнем углу

### На пилоте (для Техинкома) — без пароля

В пилотном режиме **пароль не используется** — это on-premise система внутри
завода, пользователи переключают роль через UI (под кнопкой "Переключить роль"
в правом верхнем углу). В будущем добавим LDAP/AD-аутентификацию.

### 3 быстрые кнопки (для показа клиенту/руководству)

На главной странице есть синяя плашка **"🎬 Показ клиенту — быстрая смена роли"**
с тремя большими кнопками:

- **👨‍🔧 Технолог** — создаёт ТК
- **👑 Гл. технолог** — утверждает ТК
- **🛡 Админ** — управляет системой

Нажмите кнопку — страница перезагрузится, и **цветной badge в header покажет вашу роль**.

### Селектор ролей (для всех 7 ролей)

В header справа — выпадающий список со всеми 7 ролями:

- 👨‍🔧 Технолог
- 👑 Гл. технолог
- 📏 Нормировщик
- 📐 Конструктор
- 🏭 Нач. цеха
- 🔍 ОТК (контролёр)
- 🛡 Админ

### Текущая роль

В header справа от селектора — **цветной badge с иконкой и названием роли**:

- Синий = Технолог
- Фиолетовый = Гл. технолог
- Красный = Админ
- Голубой = Нормировщик
- Оранжевый = Нач. цеха
- Зелёный = Конструктор
- Розовый = ОТК

**Всегда видно, в какой роли вы сейчас работаете.**

## 1. 👨‍🔧 Технолог (основная роль) — создаёт ТК

**Главный пользователь системы.** Создаёт техкарты.

### Что вы можете

✅ Создавать новые детали (＋ Новая деталь)
✅ Запускать AI-генерацию ТК (🤔 Уточнить → ⚡ Draft → ✨ Полная ТК)
✅ Редактировать операции (название, оборудование, время, материалы)
✅ Добавлять, удалять, восстанавливать операции
✅ Просматривать версии (v1, v2, v3) и сравнивать их (diff)
✅ Задавать правила для AI
✅ Утверждать ТК
✅ Возвращать в работу (reopen) утверждённую ТК
✅ Видеть все seed-детали Техинкома
✅ Импортировать детали (JSON, Excel, PDF)
✅ Экспортировать в PDF, Excel, 1С:CSV
✅ Печатать с QR-кодом

### Что вы НЕ можете

❌ Видеть админ-панель (`/admin` — 403)
❌ Видеть настройки системы (`/admin/settings` — 403)
❌ Управлять другими пользователями

### Типичный workflow (15-30 минут на деталь)

1. **Главная → ＋ Новая деталь** (большая зелёная кнопка) — заполнить форму
2. **AI-помощник** (открыт по умолчанию для новой детали):
   - **🤔 Уточнить** — AI задаст 3-5 вопросов → ответьте
   - **⚡ Draft** — короткий маршрут, 3 операции, ~1₽ (5-10 сек)
   - **✨ Полная ТК** — полный маршрут с учётом ответов, ~3₽ (30-60 сек)
3. **Проверьте** операции. Кликните на название/оборудование/время — отредактируйте inline
4. **Проверьте экономику** во вкладке `💰 Экономика`: ставка, материал, накладные
5. **Утвердите:** ✅ Утвердить (большая зелёная кнопка справа) → галки в чеклисте → подтвердить
6. **Экспортируйте:** кнопка `📤 Ещё ▾` → PDF, Excel, CSV для 1С

### Полезные фичи

- **`localStorage` answers** — ваши ответы на вопросы AI сохраняются в браузере
- **Self-timer** — время работы с карточкой считается автоматически
- **Pre-approve checklist** — 4 галочки, не дают утвердить ТК пока не проверено всё
- **Soft-delete** — удалённые операции попадают в "Удалённые", можно восстановить

## 2. 👑 Гл. технолог — утверждает ТК

**Контролирует качество ТК и подписывает.**

### Что вы можете

✅ **Всё, что технолог** (создавать, редактировать, генерировать)
✅ **Утверждать как гл. технолог** — после того как технолог утвердил проект
✅ **Возвращать в работу** утверждённые ТК
✅ **Видеть все детали** (свои + чужие)
✅ **Видеть** историю изменений
✅ **Видеть** отчёты пилота

### Что вы НЕ можете

❌ Видеть админ-панель
❌ Менять глобальные настройки

### Типичный workflow

1. **Главная** → кликни `🟡 Проектов` (KPI-карточка) — список деталей, ждут твоего утверждения
2. **Открой деталь** → просмотри маршрут, экономику, замечания
3. Если сомнения — **`↩️ Вернуть в работу`** → технолог получит уведомление
4. Если всё хорошо — **`👑 Утвердить как начальник`** (в dropdown `📤 Ещё`) → ТК уходит в архив
5. **Экспортируйте** в PDF (с QR) для архива

### Совет

KPI-карточка `🟡 Проектов` на главной — кликабельная, ведёт прямо на отфильтрованный список.

---

## 3. 🏭 Нач. цеха — финальное согласование

**Контролирует реализуемость ТК в цеху.**

### Что вы можете

✅ **Просматривать** все утверждённые ТК
✅ **Утверждать как начальник цеха** (после гл.технолога, для final-approve в 1С)
✅ **Просматривать** версии и историю
✅ **Сравнивать** с эталонными бенчмарками

### Что вы НЕ можете

❌ Создавать детали
❌ Генерировать ТК
❌ Редактировать операции
❌ Видеть админ-панель

### Типичный workflow

1. **Главная** → кликни `🟢 Утверждено` (KPI) — финальные ТК
2. **Открой** деталь → проверь, что:
   - Оборудование есть в вашем цеху
   - Время реалистично
   - Материалы доступны
3. **`👑 Утвердить как начальник`** (в dropdown `📤 Ещё`) или **`↩️ Вернуть в работу`**

## 4. 🛡 Админ — управление системой

**Единственная роль с доступом к админ-панели.**

### Что вы можете

✅ **Всё, что технолог + гл.технолог**
✅ **Доступ к `/admin`** (8 страниц через dropdown в header):
  - `/admin` — главная админки
  - `/admin/users` — пользователи
  - `/admin/login-log` — кто когда входил
  - `/admin/llm-calls` — все вызовы LLM, цена, ошибки
  - `/admin/settings` — настройки (YandexGPT, Telegram, SMTP)
  - `/admin/errors` — последние 50 ошибок
  - `/admin/system` — состояние системы
  - `/admin/backup` — ручной бэкап
✅ **Резервное копирование** (ежедневно 3:00, retention 30 дней)
✅ **Health-check** — мониторинг каждые 5 минут

### Типичные задачи

#### Добавить YandexGPT API key
1. Переключитесь в роль **Админ** (header справа)
2. **Настройки** (`/admin/settings`) → группа **LLM (YandexGPT)**
3. Введите API key → `💾 Сохранить`
4. Система **проверит ключ** через YandexGPT API перед сохранением

#### Посмотреть, что LLM тратит
1. **LLM-вызовы** (`/admin/llm-calls`) — таблица всех вызовов
2. Столбцы: время, деталь, модель, токены, стоимость, ошибка
3. Фильтр по дате, по детали, по статусу

#### Скачать бэкап
1. **Бэкап** (`/admin/backup`) → `💾 Скачать .db`
2. Или через CLI: `cd /opt/beget/bit-technolog && bash backup.sh`

#### Обновить систему
1. SSH: `ssh user@server`
2. `cd /opt/beget/bit-technolog && bash deploy.sh`
3. Скрипт: git pull → pytest → restart → verify

Полный гайд — см. [`12-admin-guide.md`](12-admin-guide.md).

## Частые сценарии

### "Хочу посмотреть, как работает утверждение ТК"

1. Главная → `＋ Деталь` → создайте любую
2. Откройте её → AI-помощник → `⚡ Draft`
3. Дождитесь (5-10 сек в demo, 30-60 сек в LLM)
4. `✅ Утвердить` → галки в чеклисте → `✅ Да, утвердить`
5. Теперь переключитесь в роль **Гл. технолог** (через селектор)
6. Откройте ту же деталь → появится кнопка `Утвердить как гл. технолог`
7. Нажмите → статус станет `🟢 Утверждён`

### "Хочу импортировать список деталей из Excel"

1. Подготовьте файл: `id, designation, name, model, chassis, material, size_mm, mass_kg, surface_treatment`
2. Главная → `＋ Деталь` → `Импорт` (в будущем) или через API:
   ```bash
   curl -X POST http://server:8081/api/import/tk \
     -F "file=@details.csv"
   ```
3. Система создаст детали, проиндексирует в RAG

### "Хочу посмотреть отчёт пилота"

Главная → `📈 Отчёт` (`/pilot/report`) — сводка за неделю/месяц:
- Принято/отклонено ТК
- Стоимость LLM за период
- Acceptance rate (% принятых draft'ов)
- Время на деталь (медиана)

## Что нового в M22 (v0.4.18) — полный редизайн UI

M22 — это **полный визуальный редизайн** в стиле единой дизайн-системы.

### Что изменилось

1. **Единый `design-system.css`** — все стили вынесены в один файл (`/static/design-system.css`).
   Раньше CSS был размазан по `style.css` + inline-стилям в шаблонах. Теперь один источник.
2. **CSS-переменные** — `--c-brand`, `--c-bg`, `--c-text-muted` и т.д. Меняя одну переменную,
   меняется вся цветовая схема.
3. **БЭМ-подобные классы** — `.kpi-card.kpi-new`, `.btn.btn-primary.btn-lg`, `.role-chip[data-role]`.
   Больше нет `<div style="color:red; font-size:14px;">`.
4. **Header стал sticky** — при прокрутке страницы меню остаётся сверху.
5. **Drop-down меню** — «Справочники» и «Отчёты» свёрнуты в dropdown (раньше — 6 ссылок
   в header). Сэкономили ~40% горизонтального места.
6. **Быстрая смена роли** — на главной 3 большие карточки (Технолог/Гл.технолог/Админ).
   Текущая роль выделена цветом.
7. **KPI-карточки кликабельные** — клик на «🟡 Проектов» ведёт прямо на `/?status=draft`.
   Раньше надо было фильтровать через dropdown.
8. **Detail-страница** — убраны 9 свёрнутых `<details>`. Секции (RAG, альтернативы,
   ресурсная спецификация, связи, чертёж) — компактные карточки в одной колонке.
9. **Главная кнопка одна** — на карточке детали больше нет 11-кнопочного action-bar.
   Одна большая ✅ (или 🤖 для new) + dropdown «📤 Ещё ▾» со всеми остальными.
10. **Help внутри продукта** — страница `/help` использует те же CSS-классы,
    что и остальной UI. Раньше была стихийная смесь из inline-стилей.

### Для кого это важно

- **Технолог и Гл.технолог** — увидите редизайн сразу, на главной и в карточке детали.
- **Админ** — `/admin/*` пока что на старых стилях (выглядит нормально, но не unified).
  В Sprint 5 переведём на design-system.
- **Нач. цеха** — увидите редизайн, но кнопок у вас по-прежнему мало.

### Если что-то сломалось

Версии 0.4.x в `git`. Откатиться:
```bash
cd /opt/beget/bit-technolog && git log --oneline -5
git checkout v0.4.17
systemctl restart bit-technolog
```
