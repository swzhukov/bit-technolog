> Дата: 2026-07-19. Версия: v0.4. Основа: разбор blueprint.io (фичи UX) + разбор мирового рынка (RAG, few-shot, ролевая модель, agent pipeline, BYOK, on-prem, process-based pricing). Цель: зафиксировать, что мы делаем, для кого, как устроен продукт, и как идём от прототипа к пилоту с Техинкомом.

## 1. Позиционирование (Osterwalder Business Model Canvas)

### 1.1. Customer Segments (КТО покупает)

| Сегмент | Размер в РФ | Боль | Кто принимает решение | Кто пользуется |
|---|---|---|---|
| **Заводы спецтехники** (пожарные машины, военная, нефтегаз) | ~150 заводов, цех = 50-500 чел | 4-8 ч на техкарту, ошибки в нормах, потеря тендеров из-за скорости | Директор / главный инженер / зам по производству | Главный технолог + 2-5 технологов |
| **Заводы металлообработки** (ОПК, машиностроение) | ~3000 заводов, цех = 20-200 чел | Те же + нет единой базы знаний, уход экспертов | Главный технолог | Технологи + нормировщики |
| **Проектные институты** (КБ машиностроения) | ~500 КБ | Долго от КД до ТП, расчёт себестоимости на коленке | ГИП / Главный конструктор | Конструкторы + технологи |

**Наш primary сегмент**: первый. Пожарные машины = наш anchor, спецтехника = расширение.

**Сознательно НЕ берём** (пока):
- Hobbyist / makers (это blueprint.io territory)
- Hardware/electronics (Zoo.dev, AdamCAD)
- Чистый PLM/MES-wide (Siemens, PTC, Dassault) — мы niche, не платформа
- Predictive maintenance / QC vision — не наша доменная область

### 1.2. Value Propositions (ЧТО мы обещаем)

**Главное обещание (3 цифры, как у Modulario и GitHub Copilot):**
1. **Time-to-techcard**: с 4-8 часов до 30 мин (снижение в 8-16 раз)
2. **Точность**: ±10-15% для похожих деталей, ±25-30% для новых (RAG-метрика)
3. **Acceptance rate**: ≥30% AI-операций принимается без правки (бенчмарк NIO, GitHub Copilot)

**Дополнительные обещания:**
- **On-premise** — данные не покидают завод (ГОЗ, военная приёмка)
- **1С:ERP ready** — запись в РС, формирование отчётов
- **Учится на правках** — каждая правка = сигнал для адаптации (RAG, rules)
- **Без vendor-lock** — BYOK (YandexGPT/Claude/OpenAI на выбор клиента)

**Чего НЕ обещаем** (как Modulario и Krasa):
- Не заменяем технолога. Ускоряем.
- Не автоматизируем 100%. 30% — это уже ROI.
- Не выдаём «готовый» результат без проверки. Технолог всегда review.

|---|---|---|

### 1.4. Customer Relationships (КАК работаем с клиентом)

**На этапе пилота:**
- Пилот = 0₽ (это наше конкурентное преимущество №1 vs ИИ-Технолог от 180-450К₽)
- Регулярные демо каждую неделю (как ИИ-Технолог, но открытые)
- Личный канал с Балановым (1 точка контакта, не «поддержка по тикету»)
- 1 владелец на стороне клиента (Главный технолог = Баранов у Техинкома)

**На этапе продажи:**
- Внедрение = 1-3 мес (vs 1-6 у ИИ-Технолога)
- Onboarding в первый день (Docker-compose или .exe)
- Training 2-4 ч для технологов (не неделя)
- 30 дней поддержки в стоимости

**На этапе поддержки:**
- SLA 24ч на критичные, 72ч на обычные
- Telegram-чат с инженером (не «форма обратной связи»)
- Ежемесячный релиз с фичами (по запросу клиента)

## 2. Цели и ценности продукта

### 2.1. Миссия

**Сократить время технологической подготовки производства с часов до минут, сохранив качество и контроль технолога.**

Конкретно: каждый технолог в России тратит ~4-8 часов на одну техкарту. Это значит, что:
- Завод теряет 2-3 тендера в месяц (не успевает посчитать)
- Ошибки в нормах = штрафы от заказчика
- Уход эксперта = потеря 5-10 лет институциональной памяти

**Наша миссия** — сделать так, чтобы AI брал на себя рутину (60-70% работы), а технолог фокусировался на критичных решениях (30-40%).

### 2.2. Визия (3 года)

**Стать стандартом AI для техпроцессов в РФ-спецтехнике и смежных отраслях.**

Не «ещё один ИИ-инструмент», а **де-факто-стандарт**:
- Цитируется в отраслевых гайдах
- Включается в перечень 1С-партнёров
- Используется на 50+ заводах (прямые + интеграторы)
- Публикует open-source версию для академии

**Почему реалистично:**
- 70% AI-пилотов фейлят → барьер для новых игроков высок
- У ИИ-Технолога нет публичных кейсов с цифрами → мы можем стать reference
- 1С-франчайзи ищут AI-фичи для своих клиентов → у нас готовая интеграция
- Рынок РФ спецтехники: ~150 заводов, ~300К техкарт/год, ~2-3М человеко-часов/год → TAM ≈ 1.5М₽ × 150 = 225М₽ (без учёта 1С-интеграции)

### 2.3. Ценности (как мы принимаем решения)

**1. Технолог — главный, AI — помощник**
- Любая фича спрашивает: «это помогает технологу или заменяет его?»
- Если заменяет — пересматриваем
- AI никогда не approve'ит сам. Только технолог или начальник.

**2. Точность важнее скорости**
- 30% принятых операций с правдоподобным временем > 80% принятых с фантазиями
- Лучше сгенерировать мало, но точно
- Лучше предупредить «не знаю» чем галлюцинировать

**3. On-premise = требование, не опция**
- Без on-prem мы не попадаем в ГОЗ, оборонку, банки, страховые
- Все архитектурные решения проверяем: «а это работает on-prem?»
- LLM должен уметь работать через приватный endpoint (YandexGPT Private, Anthropic VPC)

**4. Прозрачность (audit log)**
- Каждое действие AI логируется (что спросил, что ответил, что approve'нули)
- Каждая правка = причина + автор
- Это не «наблюдение», а защита технолога («я спросил AI про X, он ответил Y, я изменил Z, потому что W»)

**5. Учимся на правках**
- Правка технолога = самый ценный сигнал
- Каждый rule_extractor работает на правках
- RAG индексирует approved черновики
- Это не «дообучение модели», а RAG + rules (что мы спроектировали в `bit-technolog-training-architecture.md`)

**6. Простота > функциональность**
- 5 фич, которые работают, лучше 50, которые глючат
- Лучше узкое решение, чем широкая платформа
- «Сделай минимум — и сделай его хорошо» (это про UX, а не про скоуп)

**7. Российская специфика = наш moat**
- ГОСТ, ЕСТД, ТУ, военные стандарты — западные конкуренты не понимают
- 1С:ERP = стандарт де-факто в РФ
- Русский язык, российские промпты, российские кейсы

|---|---|---|---|---|

### 5.4. RAG Architecture (Sprint 2)

**ChromaDB setup:**
```python
import chromadb
client = chromadb.PersistentClient(path="./.chroma")
collection = client.get_or_create_collection(
    name="tech_cards",
    metadata={"hnsw:space": "cosine"}
)
```

**Embedding pipeline:**
```python
from openai import OpenAI
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def embed(text: str) -> list[float]:
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return response.data[0].embedding

# Embed: designation + name + material + operations_summary
text = f"{designation} {name} {material} {summary}"
vector = embed(text)
```

**Retriever with hybrid scoring:**
```python
def retrieve_similar(detail, top_k=3):
    # 1. Embed query
    query_embedding = embed(detail_to_text(detail))
    
    # 2. Vector search
    vector_results = collection.query(
        query_embeddings=[query_embedding],
        n_candidates=20  # over-fetch for hybrid filtering
    )
    
    # 3. Hybrid score
    scored = []
    for result in vector_results['metadatas'][0]:
        sim = vector_results['distances'][0][i]
        mat = 1.0 if result['material'] == detail['material'] else 0.5
        eq = jaccard(result['equipment'], detail['equipment'])
        op = 1.0 if ops_in_range(result['op_count'], detail['op_count']) else 0.7
        score = 0.5 * sim + 0.3 * mat + 0.2 * eq
        scored.append((score, result))
    
    # 4. Return top-3
    return sorted(scored, reverse=True)[:top_k]
```

**Cost:**
- Indexing 575 деталей: $0.05 разово
- Retrieval: $0.001/query (small text)
- Total for pilot (50-100 queries): $0.10-0.20
