# БИТ.Технолог — UX-флоу технолога

> **Дата:** 2026-07-16
> **Назначение:** описать, что технолог видит на экране и как работает с системой
> **Контекст:** MVP v0.1 (watcher КОМПАС → LLM → web → Excel). On-premise.

---

## 0. РОЛИ В СИСТЕМЕ

| Роль | Что делает в системе | Когда |
|---|---|---|
| **Конструктор** | Сохраняет .m3d/.cdw в watch-папку. Видит свои детали. | Каждый день |
| **Технолог** (главный пользователь) | Получает черновики, правит, утверждает. Видит все детали. | Каждый день |
| **Начальник технологического бюро** | Утверждает спорные ТК, видит метрики. | Раз в неделю |
| **Админ** | Настраивает справочники, мониторит систему. | По запросу |

## 8. ДЕЙСТВИЯ В КАРТОЧКЕ

| Кнопка | Что делает | Когда доступна |
|---|---|---|
| **[Сохранить]** | Сохраняет текущую версию черновика | Всегда |
| **[Утвердить]** | Фиксирует как итоговую ТК, переводит в статус «Утверждён» | Когда все вопросы отвечены и warnings < 2 |
| **[Отклонить]** | Отправляет на доработку к конструктору (с комментарием) | Когда есть блокирующие warnings |
| **[↗ В 1С]** | Экспортирует в Excel по шаблону 1С:ERP | Когда статус «Утверждён» |
| **[Обновить из КОМПАС]** | Перепарсить .m3d/.cdw (если конструктор обновил) | Всегда |
| **[Сравнить с эталоном]** | Side-by-side с АЦ-8-40 (наш RAG-эталон) | Когда есть аналог |

## 10. УВЕДОМЛЕНИЯ

| Событие | Канал | Кому |
|---|---|---|
| Новая деталь от конструктора | Push в web + email | Технолог |
| Изменился .m3d/.cdw | Push в web | Технолог (если ТК в работе) |
| AI сгенерил черновик | Push | Технолог |
| Появились warnings (низкая уверенность) | Push | Технолог + начальник |
| Технолог оставил вопрос без ответа > 1 дня | Email | Начальник |
| Деталь в статусе «Новый» > 4 часов | Email | Технолог + начальник |
| Утверждённая ТК не экспортирована в 1С > 1 дня | Email | Технолог |
