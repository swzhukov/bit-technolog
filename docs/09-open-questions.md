## ✅ РЕШЕНО

### 1. Кто участвует в пилоте?
**Решение:** Главный технолог (Баранов) + 2-3 рядовых технолога. Реалистично для пилота.
**В коде:** ROLES['main_technologist'] и ROLES['technologist'] + admin.

### 2. Где физически работают технологи?
**Решение:** В офисе Техинкома (не в цеху). Wi-Fi в офисе есть.
**Следствие:** Offline-режим НЕ нужен. Mobile — да, но не критично для пилота.

### 3. Search по шасси?
**Решение:** Добавил search по chassis (5 полей вместо 4).
**В коде:** `app.py:index` (q LIKE chassis).
**Коммит:** `a33bc0c`

### 4. Что считаем "принятой" техкартой?
**Решение:** Технолог утвердил после всех проверок. Узкое определение.
**В коде:** status='approved' в drafts. KPI считает так.
**Примечание:** В будущем — более гранулярное определение.

### 5. Кто первый смотрит UI?
**Решение:** Баранов + Голубев + рядовые вместе. Демо для всех.
**В коде:** `/demo` (5-мин сценарий).

### 6. Когда стартуем пилот?
**Решение:** 27 июля 2026.
**В коде:** Pilot ready (commit `abe403e`).

### 7. Целевой % принятия?
**Решение:** 30% (industry standard, GitHub Copilot).
**В коде:** `LLM_DAILY_COST_LIMIT_RUB=200` в .env.vps.example, `accepted_target: 30` в pilot metrics.

### 8. Целевое время на техкарту?
**Решение:** 60 мин (реалистичный, не 30).
**В коде:** `time_target: 60` в pilot metrics.

### 9. Формат исторических ТК?
**Решение:** Все форматы — Excel/PDF/Word/JSON/сканы. Универсальный импорт.
**В коде:** `importers.py` (4 формата + drawings).

## ❓ Нерешённые вопросы (новые, требуют обсуждения)

### 15. Кто именно будет пользователем-админом в Техинкоме (Сергей) — будет ли он удалённо или локально?
**Контекст:** Если Сергей админ — он должен иметь доступ к VPS, не только к Beget. Нужен SSH-доступ или web-интерфейс.
**Возможные решения:**
- A: Сергей настраивает через `/admin/settings` (уже работает)
- B: Сергей заходит через SSH и правит `/opt/beget/bit-technolog/.env`
- C: Дать Сергею отдельного admin-пользователя с расширенными правами

### 16. Что делать с деталью, если LLM выдаёт таймаут (YandexGPT недоступен)?
**Контекст:** В текущей реализации `/api/generate` вернёт ошибку. Технолог не получит проект ТК.
**Возможные решения:**
- A: Retry с exponential backoff (3 попытки, 1с/2с/4с)
- B: Fallback на Claude через OpenRouter
- C: Показать "AI временно недоступен" + кнопка "Создать ТК вручную"
- D: Показать последний кешированный draft (если был)

### 17. Как обрабатывать «откат» версии проекта ТК?
**Контекст:** Текущий diff view (v1 vs v2) есть, но нет кнопки «Откатить к v1».
**Возможные решения:**
- A: Кнопка «Откатить к v1» (требует API endpoint)
- B: В истории правок — список версий, каждая кликабельна
- C: Копирование операций из старой версии в новую

### 18. Как мерять «улучшение со временем» (RAG-learning эффект)?
**Контекст:** Мы говорим «AI учится на правках», но это не измеряется. Acceptance rate может расти или не расти.
**Возможные решения:**
- A: Считать acceptance rate по неделям (week 1, week 2, week 3, week 4)
- B: Считать time-to-card по неделям
- C: Сравнивать среднее количество правок на проект ТК
