# M38: Аудит по 5 точкам зрения (БИТ.Технолог v1.0.0, 22.07.2026)

## 1. Цели и ценности (Goals & Values)

### Заявленные цели (из README)
- Генерировать черновики техкарт за 30-60 минут вместо 4-8 часов
- AI-помощник технолога для ПК «Техинком-Центр»
- Рабочая система для 50+ технологов

### Реальность
- ✅ Скорость: RAG = 30мс, RAG+LLM = 24 сек (M35r)
- ✅ 51 изделие, 15 эталонов в БД
- ⚠️ 1bitai.ru (deepseek-v4) — внешний LLM, single point of failure
- ❌ **Не проверялось на 50+ юзерах под нагрузкой реальной работы** (load test только smoke)

### Соответствие ценностям
| Ценность | Соответствие |
|----------|--------------|
| Скорость | ✅ 24 сек |
| Качество | ⚠️ 9% зелёных норм — низко |
| Справедливость | ✅ Rate limit (5/мин) защищает от злоупотреблений |
| Прозрачность | ⚠️ "Светофор" заменили, но остались "v12 draft", "REF_1C" |
| Удобство | ⚠️ Inline-edit хорош, но 5 вкладок + 8 блоков в карточке = перегруз |

### Находки
- **GV-1:** Цель "30-60 минут вместо 4-8 часов" не измерена end-to-end (только LLM 24 сек)
- **GV-2:** "9% зелёных" — нужно понять что это значит для пользователя
- **GV-3:** "Пилот 27.07" — 5 дней. Достаточно ли времени для обучения?

## 2. Концепции, фреймворки, механизмы (Concepts & Architecture)

### Архитектура
- ✅ Clean architecture: domain / gateways / services / repositories
- ✅ FastAPI + Jinja2 + SQLite
- ✅ RAG (TF-IDF + cosine) для поиска аналогов
- ✅ LLM (1bitai.ru / deepseek-v4) для refine
- ✅ 4 роли (admin / main_technologist / technologist / workshop_chief)
- ✅ Детерминированная РС-фабрика (8 осей)

### Концептуальные вопросы
- **CA-1: items — деталь/сборка/покупное**
  - Решение: 1 таблица items с level (detail/assembly/product/purchased/semi)
  - Плюс: гибкость
  - Минус: для технолога "уровень" — внутренний термин
- **CA-2: sourcing = make/buy/coop_da/coop_full**
  - Плюс: явная модель кооперации
  - Минус: пользователь видит "Кооперация (давальч.)" — что такое "давальческое"?
- **CA-3: edit → etalon → RAG петля**
  - В Sprint 5 ADR-0011 была идея
  - Реализована в api_approve (INSERT в etalons)
  - **Не работает end-to-end:** правки (edits) НЕ попадают в etalons автоматически
- **CA-4: "светофор" — evidence-based нормы**
  - green (подтверждено) / yellow (похоже на правду) / red (догадка AI)
  - **Концепция сильная**, но "догадка AI" звучит неуважительно к AI
  - **9% green** — это плохо или нормально для пилота?

### Фреймворки
- ✅ Playwright (E2E) — 44/44
- ✅ Locust (load) — 22.7 req/s
- ✅ FastAPI + uvicorn (production)
- ✅ SQLite (WAL mode)
- ⚠️ Fernet encryption для LLM API keys
- ❌ Нет OpenTelemetry / Sentry (нет мониторинга в проде)

### Находки
- **CA-5: петля правок (edits → etalons) не автоматизирована** — критично для самообучения
- **CA-6: "Кооперация (давальч.)" — незнакомый термин** для технолога
- **CA-7: "Догадка AI" — плохой язык** для AI-assist системы
- **CA-8: 8 осей РС — пользователь не знает что это** ("Шаблоны маршрутов" + lead помог, но глубже не описано)
- **CA-9: RBAC — есть в коде, но не во всех UI местах** (A21, кнопки в /detail)

## 3. Реализация (Implementation)

### Код
- ✅ Чистая архитектура, типизация
- ✅ Whitelist полей в inline-edit (SQL-injection защита)
- ✅ CSRF middleware
- ✅ Rate limiter
- ✅ LLM semaphore
- ⚠️ In-memory `_sessions` (известно)
- ⚠️ In-memory `_rate_limit_buckets` (известно)
- ⚠️ In-memory `_llm_semaphore` (известно)

### Тесты
- ✅ Unit: 71/78 passed (3 fail — TestEvidence, TestNotices — старые)
- ✅ E2E: 42-44/44 (флаки)
- ✅ Load: 22.7 req/s, 0 5xx

### Находки
- **IM-1: флаки в E2E (S04 buy, S05 inline-edit)** — race condition
- **IM-2: 3 unit-теста падают стабильно** — TestEvidence, TestNotices. **Не исправлено!**
- **IM-3: `edits` записываются, но НЕ влияют на `etalons`** — петля обратной связи не полная
- **IM-4: парсер ТП (OCR) — 5 из 8-10 операций** (M34) — для пилота не критично, но странно
- **IM-5: `1bitai.ru` rate limit** — 5 одновременных через semaphore, но при >5 в очереди они ждут. Если 50 юзеров нажмут одновременно, последний ждёт 50×24сек = 20 мин!
- **IM-6: LLM стоит 1.32₽ за refine** — для пилота 50 юзеров × 5 генераций/день = 330₽/день = 10К₽/месяц. Нужно отслеживать.

## 4. UX и юзабилити для машиностроения (UX for Manufacturing)

### Кто пользователь?
- **Технолог** (50+ в Техинком-Центр) — создаёт ТК, знает 1С
- **Гл. технолог** — утверждает, контролирует качество
- **Нач. цеха** — получает наряд, контролирует выполнение
- **Admin** — настраивает LLM, эталоны

### Контекст использования
- Технолог сидит за компом, открывает 1С, потом БИТ.Технолог
- Видит чертёж детали (PDF/DWG) + спецификацию
- Генерирует ТК → правит → утверждает → отправляет в 1С

### UX-проблемы

| Проблема | Серьёзность | Фикс |
|----------|--------------|------|
| "v12 (draft)" в Доп.параметры | 🟡 | Переименовать в "Черновик v12" |
| "v12" badge "Черновик v12" | 🟡 | Должно быть "Черновик #12" или "Черновик 12" |
| "Догадка AI" | 🔴 | "По аналогии" / "Предположение AI" |
| "0 подтверждено, 5 похоже на правду" | 🟡 | Числа без контекста. Что значит 5? Из 5 операций? |
| "REF_1C: —" | 🟡 | Внутренний идентификатор, технологу не нужен |
| "v12 draft" в Доп.параметры | 🟡 | Не нужно в карточке (есть в badge) |
| "Слабый аналог. Технологу рекомендуется проверить" | 🟡 | Слишком длинно для столбца |
| "5 похоже на правду" | 🟡 | Технолог не понимает термин "светофор" |
| Кнопка "Аналоги" / "Подтвердить" в каждой строке | 🟡 | Что они делают? Tooltip нужен |
| 8 блоков на странице детали | 🟡 | Перегруз — нужны tabs или collapse |
| "Эталоны в базе знаний (5)" внутри карточки | 🟡 | Не относится к детали, общая база |
| "Доп. параметры" — материал/модель пусто | 🟡 | "—" в UI выглядит как ошибка |
| "v12 (draft)" повторяется в badge + в Доп.параметры | 🟡 | Дублирование |
| 24 сек ожидания без progress bar | 🔴 | UX-критично |
| "Тарлецкий А.С." автор ТК | 🟡 | ПДн, но не критично (внутри компании) |

### UX-фиксы приоритет
1. **24 сек progress bar** — критично
2. **"Догадка AI" → "Предположение AI"** — точечно
3. **Убрать дублирование "v12"** — точечно
4. **Tooltip на "Аналоги"/"Подтвердить"** — точечно
5. **Свернуть "Эталоны в базе знаний"** — структурно
